git dan github

vcs: version control sistem
merupakan perangkat lunak yang digunakan oleh pengembang perangkat lunak untuk melacak dan mengelola perubahan pada kode proyek, sehingga mereka dapat kembali ke versi sebelumnya jika diperlukan.
git
sistem kontrol versi terdistribusi yang dirancang untuk melacak perubahan dalam file, terutama kode sumber, sehingga pengembang dapat mengelola riwayat proyek dan berkolaborasi secara efisien. Fitur utamanya termasuk kemampuan untuk melacak perubahan, kembali ke versi sebelumnya, membuat cabang untuk pengembangan fitur baru tanpa mengganggu kode utama, dan bekerja secara offline karena setiap pengembang memiliki salinan lengkap dari repositori. salah satu software yang bisa melakukan vcs.
github
Platform ini menggunakan Git untuk menyediakan kontrol versi terdistribusi, sementara GitHub sendiri menyediakan kontrol akses, pelacakan bug, permintaan fitur perangkat lunak manajemen tugas, integrasi berkelanjutan. versi mudahnya adalah, salah satu website / layanan untuk mengelola project git, dan layanan cloud

istilah dalam git

  1. repository : tempat penyimpanan project git
  2. commit : menyimpan perubahan pada project git
  3. hash : penanda unik pada sebuah commit
  4. checkout : berpindah ke sebuah commit
  5. branch : cabang dari sebuah repository
  6. merge : menggabungkan branch ke branch utama (main/master)
  7. remote : sumber yang memiliki repo
  8. clone : menyalin repository dari github ke local
  9. pull : mengambil perubahan dari repository github ke local
  10. push : mengirim perubahan dari local ke repository github
