yang pertama kita buat folder program kita dan membuat file didalam folder tersebut. untuk editor seperti vscode bisa buat langsung didalam console terminal untuk bisa mengobprasikan git bash

jika kita sudah membuat file dalam format .html menjadi halaman sederhana kemudian kita save, ketika kita sedang mengerjakan sebuah project, tambah file, hapus file atau mengedit isi sebuah file lalu saya save, itu hanya berlaku untuk file sistem saya saja, seperti ketika saya sedang menyimpan sebuah project. buat git nya belum berlaku. supaya gitnya mengenali perubahan nya kita harus commit dulu, dan sebelum commit ada hal yang harus dilakukan

pastikan kita kembali pada git bash (untuk yang menggunakan sublime text) namun untuk code editor sepertis VSC kita hanya perlu membuka nya pada terminal dan memilih powershell untuk program deafult pada vsc, atau menggunakan gitbash. untuk kali ini saya menggunakan gitbash


$ ls
ls itu list isi foldernya, ngelist ada file apa saja.

lalu jika menuliskan $ git status nanti git memberitahu kita ada file baru apa, atau ada file yang berubah yang mana yang belum kita commit. jika kita enter, git akan memberikan status nya deperti file yang masih untracked , belum di commit, sudah di commit dan lain sebagainya

jika ada file yang belum di tracked contoh index.html, file tersebut harus kita simpan kedalam stagging area. jadi tidak bisa asal langsung simpan.




dalam pengaplikasian git kita harus mwnggunkan init untuk menambahkan file program kita ke daalam repositosi kosong pada git

$ git add untuk menambahkan file ataupun perubahan pada file kita. contoh jika kita melakukan perubahan pada file style.css maka kita akan memasukan kunci nya $ git add style.css. berlaku pada file lain dengan menulikan nama file tersebut

tetapi jika kita ingin menambahkan semua file tersebuk kita gunakan $git add . tanda titik pada akhir kalimat add tersebut digunakan untuk menambahkan semua file yang ingin kita ubah

jika kita memodify file, metelah melakukan modify lakukan (add .) lalu siap di commit isi pesan nya sesuai dengan ketentuan yang ada buat



sekarang setiap perubahan yang dilakukan di source code yang sedang digunakan, tidak selesai hanya dengan di save saja, harus juga dilakukan dengan commit,kalau pada github dia bisa save langsung otomatis commit. kalau di lokal harus tertib, kalau sudah disave, harus di commit

sebelum commit pastikan untuk sudah config --global, dengan diisi username lalu alamat email yang digunakan untuk menghubungkan akun github

jika kita ingin melihat apa saja yang pernah kita lakukan kita bisa tulis $ git log

jika kita ingin melihat 3 commit sebelum nya $ git log -3

atau mau melihat yang spesifik, commit mana yang terjadi untuk sebuah file tertentu (contoh ketika kita menghapus file style.css dan hilang) $ git log (gunakan parameter tambahan -- untuk menemukan sebuah file) --style.css

misal kita telah menghapus file nya terus kita butuh file nya, tetapi di folder sudah tidak ada, tepi sebenarnya didalam git tersebut file nya masih ada, dan kita bisa meminta .git untuk mengembalikan file tersebut contoh nya file style.css dengan checkout kedalam keadaan saat sebelum di hapus dengan menulsikan $git checkout (lalu 5 digit pertama pada commit hash nya) --style.css
